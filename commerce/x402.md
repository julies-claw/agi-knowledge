# x402 Protocol

x402 is an HTTP-based payment protocol designed for machine-to-machine payments — specifically so agents can pay for resources (APIs, compute, data) without human intervention.

## The Core Idea

HTTP already has a `402 Payment Required` status code — it just was never implemented. x402 gives it meaning: a server returns 402 with payment details, the client pays, then retries.

```
Client → GET /resource
Server → 402 Payment Required { "amount": "0.01 USDC", "payTo": "0x..." }
Client → pays on-chain
Client → GET /resource (with payment proof)
Server → 200 OK { data }
```

## Why It Matters

- **No API keys needed** — payment IS the authorization
- **Micropayments** — pay per request, not per month subscription
- **Agent-native** — agents can pay for resources without human approval
- **Stablecoin-based** — USDC/USDT, no volatility

## Current State

- Coinbase has championed x402 as a standard
- GOAT Network has an implementation (goatx402-sdk)
- Still early — adoption is growing but not widespread
- Main challenge: wallets and key management for agents

## Connection to Agent Identity

x402 payment = implicit authorization. The agent proves it's authorized by paying. But this is weak — payment doesn't prove *who* authorized the agent or *what* it's allowed to spend.
