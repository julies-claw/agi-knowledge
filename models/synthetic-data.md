# Synthetic Data — Training AI on AI-Generated Data

As real human-generated data runs out (or becomes legally contested), AI labs are increasingly training on data generated by AI itself. This has profound implications.

## Why Synthetic Data

- **Real data scarcity**: web-scale text largely consumed; new human writing can't keep up
- **Quality control**: synthetic data can be generated to spec (specific skills, difficulty levels)
- **Privacy**: generate training data without exposing real user data
- **Cost**: generating data is cheaper than collecting and labeling it

## Current Use

- **Instruction tuning**: generating synthetic instruction-response pairs at scale
- **Reasoning**: generating chains of thought, math solutions, code with tests
- **RLAIF**: Claude generates AI feedback to replace (or supplement) human feedback
- **Distillation**: teacher model generates data → train smaller student model

## The Model Collapse Risk

If you train on AI-generated data, then train the next model on that model's output, and so on:
- Errors and biases compound
- Diversity decreases over generations
- Models converge toward the same patterns

Research (arXiv 2410.16713): theoretical analysis of "collapse or thrive" scenarios in self-generating training loops.

Observed in practice: models trained on LLM output tend to produce more homogeneous, formulaic text.

Source: Transparency Coalition, "Synthetic Data and AI Model Collapse" (Oct 2025); arXiv paper (Mar 2025)

## The Counter-Argument: Data Flywheels

Leading labs use proprietary feedback loops:
- User interactions generate implicit labels
- Model outputs rated by humans → reinject
- Synthetic data filtered by quality models

The question isn't "real vs. synthetic" but "high-signal vs. low-signal" data.
